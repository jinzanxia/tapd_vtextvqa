"""
HIERARCHICAL EVIDENCE MINING PIPELINE
├── Stage 1: Question Structural Parsing
│   └── parsing/question_parser.py
│       - Uses Qwen2.5-VL to parse questions into structured format
│       - Extracts: target, attribute, relation, task
│       - Example: "What does the blue sign say?" → 
│         {"target": "sign", "attribute": "blue", "relation": "", "task": "ocr"}
│
├── Stage 2: Frame-Level Relevant Frame Retrieval  
│   └── retrieval/frame_retrieval.py
│       - Retrieves top-K frames using VLM relevance scoring
│       - Uses coarse-grained retrieval prompt
│       - Example: "Does this frame contain a blue sign?"
│       - Output: [{"frame_id": 0, "frame": PIL.Image, "score": 0.95}, ...]
│
├── Stage 3: Target Region Localization
│   └── retrieval/region_localization.py
│       - Localizes target regions in retrieved frames
│       - Uses object-centric grounding prompts
│       - Example: "Locate the blue sign"
│       - Output: Candidate regions with bboxes and scores
│
├── Stage 4: OCR Visibility Scoring
│   └── retrieval/ocr_visibility.py
│       - Evaluates crop quality for OCR readability
│       - Combines three metrics:
│         * OCR confidence (PaddleOCR) - weight: 0.4
│         * Sharpness (Laplacian variance) - weight: 0.3
│         * VLM visibility (Qwen2.5-VL) - weight: 0.3
│       - Selects best crop for OCR tasks
│
├── Stage 5-6: Evidence Fusion & VLM Reasoning
│   └── reasoning/qwen_reasoning.py
│       - Combines global frame + local crop
│       - Uses Qwen2.5-VL for dual-image reasoning
│       - Generates final answer
│
├── ORCHESTRATOR
│   └── pipeline/evidence_pipeline.py
│       - EvidenceMiningPipeline class
│       - Coordinates all 6 stages
│       - Provides run() method for end-to-end inference
│
└── UTILITIES
    └── utils/prompt_builder.py
        - Constructs optimized prompts for each stage
        - Functions for each pipeline component
"""
